* 논의 내용


1. 강의실 구현 방법 결정

후보 1) 로 결정

강의실 시간 및 요일 별로 Up Table, Down Table 만들기

*Up Table : 시간대별로 1층에서 올라가는 인원

*Down Table : 시간대별로 x층(x >1)에서 내려오는 인원


2. 구체적인 문제 해결 방법 결정

Base Problem (최적의 경로 찾기) + Additional Problem (이동수단 비효율 찾기 + 임의의 위치에 이동수단 설치시 효율적인 위치 찾기)

후보 1)에 해당하는 Up Table과 Down Table을 만들어, 시간대별로 이동수단마다 붐비는 인원을 구할 수 있다.

이때 강의가 끝나는 사람들은 강의실에서 가장 가까운 엘리베이터를 타고 간다고 가정하며, 강의가 끝나는 모든 인원이 1층으로 가야할 때 시간대별로 이동 수단의 걸리는 시간을 구한다. (Down Table 관련)
또한, 1층에서 강의실로 가야하는 모든 인원이 x(x>1)층으로 갈 때 시간대별로 이동 수단의 걸리는 시간을 구한다. (Up Table 관련)

결과로 나타나는 Table을 바탕으로 비효율적인 부분을 분석한다.
모든 사람들이 이동하는데 걸리는 시간이 줄어드는 것이 효율적이라고 판단.

그리고 임의의 위치에 이동수단 설치시 사람들이 이동하는데 걸리는 시간이 줄어들 것이고, 각각의 위치마다 걸리는 시간이 줄어드는 정도를 분석해 시간이 가장 많이 줄어드는, 즉 가장 효율적인 위치를 찾는다.

또한, 이 데이터들을 바탕으로 최적의 경로를 찾아준다. (이 부분에 대한 구현 방법은 좀 더 논의 필요)

ㄱ. 엘리베이터를 탈 수 있는 확률, 못 탈 확률 등을 안내하면서 유저가 경로 선택하게 해주기.

ㄴ. 시간을 기대값으로 계산해 최적의 경로 찾아주기.
